# Pending Implementation: Deterministic Fit Score Architecture

This document tracks the pending implementation of the "Fit Score" feature for the VivaTech 2026 demo, strictly adhering to the "Glassbox AI" (EU AI Act compliant) requirement. 

**Core Principle:** *The system calculates the math deterministically; the LLM explains the reasoning.*

## 1. Data Prerequisites (Solving the Measurement Gap)
To calculate a deterministic body fit (e.g., "shoulder width matches at 91%"), we must have physical measurement data for both the user and the garments. 
- **User Measurements:** Utilize the existing `userMeasurement Json?` field on the `User` model to store the user's physical dimensions (e.g., `{ "shoulderWidthCm": 45, "waistCm": 80 }`).
- **Garment Measurements:** Utilize the existing `metaData Json?` field on the `Garment` model to store the garment's flat measurements (e.g., `{ "shoulderWidthCm": 46, "lengthCm": 70 }`). 
- **Demo Seeding:** Create a seed script that populates the specific demo garments with these JSON measurements so the math works perfectly during the exhibition.

## 2. Deterministic Layer (Backend Logic)
**Target File:** `src/utils/fit-score.util.ts`
- Create a `calculateFitScore(context, garment, userMeasurements)` function.
- It will compute a total `fitScore` (0-100) using a weighted formula:
  - `weather` match (35%)
  - `body` fit (25%) -> *Calculated by comparing User's `shoulderWidthCm` against Garment's `shoulderWidthCm` + `silhouette` allowance.*
  - `wardrobe` compatibility (20%)
  - `material` suitability (20%)
- It must return an object containing the `total` score and a `breakdown` object representing the exact component values.

## 3. ChatWonder Prompt Updates
**Target File:** `src/services/shared/chat-wonder.service.ts`
- **DO NOT** instruct the LLM to calculate or invent a numeric score.
- **DO** inject the pre-computed Fit Score and its breakdown data into the LLM prompt context for the garments being evaluated.
- Instruct ChatWonder to use the provided numeric breakdown to generate a user-friendly narrative inside the `reason` string explaining *why* the score is high or low.

## 4. Response Schema Updates
**Target File:** `src/utils/parse-chatWonder-response.util.ts`
- Update the `ResolvedProduct` interface to strictly separate the numeric values from the narrative explanation string, avoiding parsing ambiguity for the UI:

```typescript
export interface ResolvedProduct {
  id: string;
  // ... other fields ...
  fitScore: number;
  fitScoreBreakdown: {
    weather: number;
    body: number;
    wardrobe: number;
    material: number;
  };
  reason: string; // The LLM-generated narrative explanation
}
```

## Verification & Consistency Requirements
To prove "Glassbox AI" consistency during the demo, the implementation must guarantee:
1. **Measurement Reaction:** Passing a garment with `shoulderWidthCm: 45` to a user with `shoulderWidthCm: 44` should yield a high body score, while passing it to a user with `shoulderWidthCm: 55` should lower the score deterministically.
2. **Idempotency:** Passing the exact same user shape, weather, and garment into the endpoint must return the exact same numeric `fitScore` every single time.
3. **Narrative Grounding:** The `reason` string generated by the LLM must explicitly reference the factors that pulled the score up or down based *only* on the hardcoded JSON measurements.
